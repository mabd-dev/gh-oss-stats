# Release Notes - v0.2.0

**Release Date:** December 24, 2025
**Codename:** Badge Generation

## Overview

Version 0.2.0 introduces **SVG badge generation** - create beautiful, customizable badges to showcase your open source contributions on GitHub READMEs, portfolios, and websites. This release adds a complete badge generation system with 4 distinct styles, 2 themes, and full CLI integration.

## What's New

### Badge Generation System

Generate professional SVG badges from your OSS contribution stats with a single command. Perfect for:
- GitHub profile READMEs
- Personal portfolios
- Project documentation
- Social media profiles
- Developer resumes

### 4 Badge Styles

| Style | Dimensions | Description | Use Case |
|-------|-----------|-------------|----------|
| **Summary** | 400×200 | Key metrics overview | README headers, profile pages |
| **Compact** | 280×28 | Shields.io style | Inline badges, status bars |
| **Detailed** | 400×320 | Top 5 contributions | Contribution showcases |
| **Minimal** | 120×28 | Project count only | Minimalist designs |

See [Badge Documentation](../BADGES.md) for visual examples and detailed usage.

### 2 Color Themes

- **Dark Theme** - GitHub dark colors (#0d1117 background, #58a6ff accent)
- **Light Theme** - GitHub light colors (#ffffff background, #0969da accent)

### Advanced Features

- **Flexible Sorting** - Sort contributions by PRs, stars, or commits
- **Configurable Limits** - Control number of contributions shown (1-20+)
- **Smart Number Formatting** - Automatic K/M suffixes (1.6K, 125.4K, 2.5M)
- **Production Ready** - 96.1% test coverage, 58 comprehensive tests

## Qick Start

### Generate Your First Badge

```bash
# Install or upgrade
go install github.com/mabd-dev/gh-oss-stats/cmd/gh-oss-stats@v0.2.0

# Generate a summary badge
gh-oss-stats --user YOUR_USERNAME --badge

# Generate a detailed badge with top 10 by stars
gh-oss-stats --user YOUR_USERNAME --badge \
  --badge-style detailed \
  --badge-theme dark \
  --badge-sort stars \
  --badge-limit 10
```

### Generate Stats + Badge Together

```bash
# Fetch stats and generate badge in one command
gh-oss-stats --user YOUR_USERNAME \
  -o stats.json \
  --badge \
  --badge-output badge.svg
```

## New CLI Flags

### Badge Generation Flags

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--badge` | bool | `false` | Enable badge generation |
| `--badge-style` | string | `"summary"` | Badge style: summary, compact, detailed, minimal |
| `--badge-theme` | string | `"dark"` | Color theme: dark, light |
| `--badge-output` | string | `"badge.svg"` | Output file path |
| `--badge-sort` | string | `"prs"` | Sort by: prs, stars, commits (detailed only) |
| `--badge-limit` | int | `5` | Max contributions (detailed only) |

### Debug Mode Flag

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--debug` | bool | `false` | Use mock API client for local testing |

## Technical Details

### Public API

```go
import "github.com/mabd-dev/gh-oss-stats/pkg/ossstats/badge"

// Generate a badge
opts := badge.BadgeOptions{
    Style:  badge.StyleDetailed,
    Theme:  badge.ThemeDark,
    SortBy: badge.SortByStars,
    Limit:  10,
}

svg, err := badge.RenderSVG(stats, opts)
```

### Mock API Client

New debug mode for local testing without hitting GitHub API:

```bash
# Fast local testing with mock data
gh-oss-stats --user test-user --debug
```

Features:
- No API rate limits
- Instant results (<1 second)
- Uses static JSON mock files
- Perfect for development and testing

Mock data location: `internal/github/mockResponses/`
